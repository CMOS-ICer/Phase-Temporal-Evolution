The functional model developed for Chapter 4 is located in the `i_ToF` folder. With MWorks 2026a installed, simply open the `i_ToF.jl` file using MWorks Syslab and click "Run"; this launches MWorks Sysplorer to execute the `i_ToF.mo` model (shown in Fig. 5 of the paper), a process analogous to calling Simulink from MATLAB. The simulation results are saved to `i_ToF_full_model_results.csv`; the iToF data presented in Table 4 of the paper is derived from this file. Additionally, the data for Fig. 6 in Chapter 4 was obtained through this model. Parameters such as object velocity, modulation frequency, starting distance, and exposure time can be directly adjusted by modifying variables like `velocities`, `f`, `d_start`, and `sim_time` within the `i_ToF.jl` file. The configurable parameters are conveniently located at the top of the file for easy editing.
Similarly, the `Do_ToF` folder contains the model developed for Do_ToF, which is derived by modifying the parameters of the `i_ToF.mo` model. Upon execution, the model outputs `Do_tof_full_model_results.csv`, which provides the Do_ToF data for Table 4 of the paper. The workflow is identical: open `Do_ToF.jl` in MWorks Syslab and click "Run" to launch MWorks Sysplorer and execute the `Do_ToF.mo` model. Parameters such as homodyne frequency, heterodyne frequency, simulation start/end velocities, velocity step size, and exposure time can be directly modified by changing variables like `fa`, `fb`, `v_start`, `v_stop`, `v_step`, and `sim_time` within the file; these configurable parameters are also conveniently located at the top of the file.
