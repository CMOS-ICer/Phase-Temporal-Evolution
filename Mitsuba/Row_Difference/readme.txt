All Python files ending in `_d` generate the output for Fig. 7(a), while those ending in `_v` generate the output for Fig. 7(b); all files are directly executable. The filenames "cube," "sphere," and "double" correspond to the three scenes in the Mitsuba experiment section of the paper (i.e., Fig. 7); upon execution, results are saved to a directory named "result_XXX_x." Parameters such as object movement speed (note: `SPEED_OBJ2` is irrelevant except in the "double" scene), initial object distance, modulation frequency, and exposure times for odd and even rows can be adjusted by modifying variables like `SPEED_OBJ1`, `OBJ1_INIT_Z`, `MOD_FREQ`, `EXPOSURE_ODD`, and `EXPOSURE_EVEN` within the files. For files ending in `_v`, the file named `depth_combined_rows_odd8ms_even80ms.npy` in the output folder contains the depth information for both odd and even rows at the current speed. To proceed, rename this file to reflect the specific speed value and place it in the "8-80" subfolder within the "Noise Model Addition and Verification" directory; then, run `Row_Diff.m` to obtain velocity calculation results incorporating the noise described in the paper—these results will be output to the "sim_output" folder located within the "8-80" directory.
